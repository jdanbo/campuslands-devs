# Instrucciones generales

Este documento explica cómo trabajar dentro de Campuslands Devs.

## Antes de empezar

1. Lee el README principal.
2. Confirma que estás en la rama `dev`.
3. Crea una rama personal para el ejercicio.
4. Abre el README del ejercicio y léelo completo.
5. Crea tu solución dentro de una carpeta propia en `resoluciones/`.

## Cómo pensar cada ejercicio

Cada ejercicio está escrito para entrenar una habilidad real. No corras a escribir código o mover carpetas. Primero responde mentalmente:

- ¿Qué problema se está simulando?
- ¿Qué datos tengo?
- ¿Qué resultado debo entregar?
- ¿Qué reglas no puedo romper?
- ¿Cómo puedo validar mi respuesta?

## Entrega

La entrega final debe quedar así:

```text
basico/<area>/ejercicio-XX/resoluciones/nombre-apellido/archivo.extension
```

Ejemplo:

```text
basico/logica/ejercicio-03/resoluciones/juan-perez/juan-perez.js
```

## Validación personal

Antes de entregar, verifica:

- El archivo abre sin errores.
- La solución está dentro de tu carpeta personal.
- Los ejemplos del README producen el resultado esperado.
- La estructura de carpetas no cambió.
- No editaste respuestas de compañeros.
- Tu commit explica qué hiciste.
